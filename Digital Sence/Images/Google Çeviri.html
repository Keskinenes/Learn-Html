$ _SERVER ["PHP_SELF"], şu anda çalıştırılan komut dosyasının dosya adını döndüren süper bir global değişkendir.

Yani, $ _SERVER ["PHP_SELF"], farklı bir sayfaya atlamak yerine gönderilen form verilerini sayfaya kendisi gönderir. Bu şekilde, kullanıcı formla aynı sayfada hata mesajları alır.

$ _SERVER ["PHP_SELF"] değişkeni bilgisayar korsanları tarafından kullanılabilir!

Sayfanızda PHP_SELF kullanılıyorsa, kullanıcı bir eğik çizgi (/) ve ardından bazı Çapraz Site Komut Dosyası (XSS) komutlarını çalıştırabilir.

Siteler arası komut dosyası oluşturma (XSS), genellikle Web uygulamalarında bulunan bir tür bilgisayar güvenlik açığıdır. XSS, saldırganların diğer kullanıcılar tarafından görüntülenen istemci sayfalarının komut dosyalarını Web sayfalarına enjekte etmesini sağlar.

Şimdi, kullanıcı "http://www.example.com/test_form.php" gibi adres çubuğuna normal URL'yi girerse, yukarıdaki kod aşağıdakilere çevrilir:

<Script> etiketinin içine herhangi bir JavaScript kodu eklenebilir. Bir bilgisayar korsanı, kullanıcıyı başka bir sunucudaki bir dosyaya yönlendirebilir ve bu dosya, genel değişkenleri değiştirebilen veya kullanıcı verilerini kaydetmek için formu başka bir adrese gönderebilen kötü amaçlı kod içerebilir.

The $_SERVER["PHP_SELF"] is a super global variable that returns the filename of the currently executing script. So, the $_SERVER["PHP_SELF"] sends the submitted form data to the page itself, instead of jumping to a different page. This way, the user will get error messages on the same page as the form. The $_SERVER["PHP_SELF"] variable can be used by hackers! If PHP_SELF is used in your page then a user can enter a slash (/) and then some Cross Site Scripting (XSS) commands to execute.
